# 06: Agentic Systems (Tools, State, Guardrails)

Goal: learn how to design tool-using, stateful AI systems that are safe and debuggable.

This module is about engineering discipline, not agent “magic”.

## Prerequisites

- [05-genai-and-llms](../05-genai-and-llms/README.md)

## Do These Lessons

- Lesson 7:
  - [`07-agentic-ai-and-applied-ai-systems-design/README.md`](../../07-agentic-ai-and-applied-ai-systems-design/README.md)

Recommended early sub-lessons (if you are new):
- focus on tool design, failure modes, and evaluation/observability
- postpone “multi-agent” until you can test and debug a single-agent system

## Completion Criteria (Self-Check)

- You can implement a tool allowlist and argument validation.
- You can explain common agent failure modes (tool misuse, loops, prompt injection).
- You can produce traces/logs that explain why a run failed.
