## What “Good” Looks Like

- A problem statement with clear constraints.
- A baseline model and a measured improvement (not just “it seems better”).
- A reproducible run log (configs, seeds, data fingerprint).
- A serving contract and a rollback plan.
- An evaluation plan for both model quality and system reliability.
