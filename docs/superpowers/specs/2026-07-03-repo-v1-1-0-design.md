# AI Engineering Zero to Mastery: v1.1.0 Release Design

Date: 2026-07-03

## Goal

Ship a beginner-first, production-oriented curriculum repository release that is:

- easy to set up and reproduce (deterministic deps, clear tracks),
- cohesive (clear entrypoints, predictable structure),
- runnable where it claims to be runnable (capstone scaffold, validation),
- maintainable (CI + local validation tooling),
- transparent (curated references + full source registry).

## Current State (Observed)

- Repository is organized as 15 lessons (`01-`..`15-`) with consistent sub-lesson structure:
  - sub-lesson `README.md`
  - `theory/*.md` + `theory/*.pdf`
  - `notebooks/*.ipynb`
- Root has:
  - `README.md`, `HANDBOOK.md`, `HANDBOOK.pdf`
  - `docs/documentation-map.md`, `docs/sources-and-references.md`
  - a GitHub Actions workflow (`.github/workflows/curriculum-ci.yml`)
  - a minimal `requirements.txt`
- The capstone template directory exists but contains placeholder-only files.

## Problems To Solve (Audit Findings)

1. **Non-deterministic setup**
   - `requirements.txt` is incomplete relative to notebook imports.
   - Learners can install the requirements and still hit missing-package errors later.
2. **Capstone not runnable**
   - `tmp_capstone_template/*` contains `# Implement ...` placeholders.
3. **CI validation is not reusable locally**
   - Integrity checks live inline inside the workflow YAML.
   - Placeholder scanning can produce false positives if a local `.venv/` is present.
4. **Beginner onboarding can be clearer**
   - Many learners need explicit learning tracks and troubleshooting guidance.
5. **Root noise**
   - Execution-failure artifacts and research artifacts are mixed into the root and can confuse beginners.

## Non-Goals

- Executing all notebooks in CI (costly, hardware-dependent, often network-dependent).
- Rewriting the entire curriculum content; focus is on high-leverage structure and reproducibility fixes.
- Adding contributor/co-author metadata.
- Renaming the repository.

## Proposed Solution (Design)

### 1) Dependency Management: uv project + lock

- Add `pyproject.toml` as the source of truth for dependencies.
- Commit `uv.lock` for deterministic installs.
- Provide optional extras so beginners can start light and add only what they need.

Dependency sets:

- Base (core): foundations + ML basics + notebook runtime
- Optional extras:
  - `dl` (PyTorch + vision)
  - `genai` (Transformers + Diffusers + datasets + PEFT + TRL)
  - `rl` (Gymnasium)
  - `serving` (FastAPI + Uvicorn)
  - `ops` (MLflow + W&B)

### 2) Beginner Docs

- Update `README.md` to:
  - define AI engineering in plain English,
  - state who it is for,
  - provide tracks (Beginner Track + Full Track),
  - explain `uv sync` + when to add extras,
  - link to capstone scaffold,
  - include a small curated references list + link to the full registry.
- Add focused docs pages:
  - `docs/start-here.md`
  - `docs/setup-and-troubleshooting.md`
  - `docs/learning-tracks.md`

### 3) Capstone Scaffold (top-level `projects/`)

- Add `projects/capstone-template/` as a runnable, testable mini-project that demonstrates:
  - data → train → eval → serve
  - reproducibility (seed + config)
  - basic quality gates (metrics saved, API contract tested)
- Keep it offline-friendly by default (use built-in sklearn dataset).

### 4) Validation + CI

- Create `scripts/validate_curriculum.py`:
  - lesson/sub-lesson structure checks
  - notebook JSON parse + required markdown sections
  - placeholder token scan excluding `.venv/`, `.git/`, and other non-curriculum dirs
  - local link validation for the root README (optionally extended later)
- Update GitHub Actions to call the validation script.
- Add a lightweight `pytest` run for the capstone scaffold (without requiring heavy extras).

### 5) Release Hygiene

- Add MIT `LICENSE`.
- Add `CHANGELOG.md` with `v1.1.0`.
- Relocate or clearly document any execution-failure artifacts so they don’t confuse beginners.

## Acceptance Criteria

- Fresh clone → `uv sync` works for base.
- `uv sync --extra serving` enables running capstone API tests.
- `python scripts/validate_curriculum.py` passes.
- `pytest` passes.
- README entrypoint + internal links work and explain next steps for beginners.

## Rollback Plan

- Revert commits in reverse order; the largest change is dependency standardization.
- Keep `requirements.txt` as a compatibility escape hatch (generated from lock) if needed.
