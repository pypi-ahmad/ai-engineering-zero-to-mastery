# Documentation Map

This repository uses a layered documentation architecture to keep learning and maintenance clean.

## 1) Root README (`README.md`)
Use this first.

Purpose:
- Understand what this curriculum is and who it is for.
- Get setup instructions.
- Navigate all lessons quickly.

## 1.5) Curriculum Path (`curriculum/`)
Use this if you are a complete beginner or want a guided path.

Purpose:
- Follow a beginner-first order (including recommended sub-lesson ordering).
- Understand prerequisites and what to skip without losing the thread.

## 2) Lesson READMEs (`<lesson>/README.md`)
Use this second.

Purpose:
- Understand lesson goals and where each sub-lesson starts.
- Jump to theory and notebook files for that lesson.

## 3) Theory Chapters (`<lesson>/<sub-lesson>/theory/*.md`)
Use this for deep study.

Purpose:
- Learn conceptual foundations, formulas, tradeoffs, and case studies.
- Prepare for interviews and design discussions.

## 4) Teaching Notebooks (`<lesson>/<sub-lesson>/notebooks/*.ipynb`)
Use this for hands-on practice.

Purpose:
- Run examples end-to-end.
- Adapt templates/checklists to your own projects.

## 5) Exercises (`<lesson>/<sub-lesson>/exercises/*.md`)
Use this to practice deliberately after running the notebook once.

Purpose:
- Reinforce key skills with small tasks and expected outcomes.
- Build “do it yourself” confidence instead of passive reading.

## 6) Handbook (`HANDBOOK.md`, `HANDBOOK.pdf`)
Use this for end-to-end revision.

Purpose:
- Read a single consolidated learning path across all lessons.
- Review representative code and real notebook outputs.

## 7) Sources Registry (`docs/sources-and-references.md`)
Use this for validation and further reading.

Purpose:
- Trace curriculum claims back to authoritative sources.
- Continue advanced self-study from official docs and syllabi.
