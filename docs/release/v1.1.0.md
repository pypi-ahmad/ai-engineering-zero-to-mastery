# Release v1.1.0 (2026-07-03)

This release focuses on making the repo **beginner-first** and **reproducible**, while keeping the curriculum structure intact.

## Audit Findings (high signal)

- Dependency setup was not deterministic and `requirements.txt` did not cover notebook imports.
- Lesson 15’s scaffold existed as placeholders, not a runnable template.
- CI validation lived inline in workflow YAML and could false-fail when `.venv/` exists.
- Root contained execution/research artifacts that could confuse beginners.

## What Changed

- Standardized dependency management with `pyproject.toml` + `uv.lock` and documented optional extras.
- Added `projects/capstone-template/` runnable scaffold:
  - offline ingestion (built-in dataset)
  - train/test splits
  - model training + artifact persistence
  - evaluation metrics
  - optional FastAPI serving + contract test
- Added `scripts/validate_curriculum.py` and updated CI to run it.
- Added beginner onboarding docs (start here, tracks, setup/troubleshooting).
- Added MIT `LICENSE` and a `CHANGELOG.md` entry.

## How To Verify

```bash
uv venv --python 3.12
source .venv/bin/activate
uv sync --frozen --group dev
python scripts/validate_curriculum.py
uv run pytest -q
```

Optional serving test:

```bash
uv sync --frozen --extra serving --group dev
uv run pytest -q
```

## Known Limitations

- Notebooks are validated structurally and syntactically, but the repo does not execute every notebook in CI.
  - Full execution is expensive and can require GPU/network.
