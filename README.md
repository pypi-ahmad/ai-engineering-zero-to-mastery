# AI Engineering Zero to Mastery

![Build](https://img.shields.io/badge/build-curriculum--ci-blue) ![Python](https://img.shields.io/badge/python-3.10%2B-3776AB?logo=python&logoColor=white) ![License](https://img.shields.io/badge/license-unlicensed-lightgrey)

**Tagline:** A production-oriented, end-to-end AI engineering curriculum covering ML, DL, GenAI, agentic systems, MLOps/LLMOps, safety, and capstone delivery.

## Problem Statement
Most learning paths teach isolated topics (ML models, LLM prompting, or deployment) but not the full engineering lifecycle from fundamentals to shipping reliable AI systems. This repository closes that gap with a single, structured path from zero to mastery, with chapter-style theory and runnable notebooks across 15 lessons.

## Approach and Design Process
Curriculum design followed a practical progression: foundations -> ML/DL -> GenAI/agents -> operations/safety -> capstone and professional practice. Content was grounded in official documentation, university syllabi, standards references, and production guidance that are explicitly cited in lesson theory files (see consolidated source registry below).

Design principles used throughout:
- Chapter-style explanations with formal definitions and decision frameworks.
- Notebook-first practice with reusable patterns, case studies, and interview Q&A.
- Production constraints emphasized (observability, governance, reliability, cost).
- Bridge content between lessons to make the path cohesive end-to-end.

## Setup and Usage

### 1) Clone
```bash
git clone <your-repo-url>
cd ai-engineering-zero-to-mastery
```

### 2) Create environment
```bash
uv venv
source .venv/bin/activate
uv sync  # if lockfile/deps are configured
# or
uv add -r requirements.txt
```

### 3) Run notebooks
```bash
jupyter lab
# or execute a notebook non-interactively
jupyter nbconvert --to notebook --execute <notebook.ipynb> --output <executed.ipynb>
```

### 4) Export theory markdown to PDF
```bash
python3 scripts/export_theory_pdf.py --src <path/to/theory.md>
```
